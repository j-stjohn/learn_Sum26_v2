# CSCI 101 — Foundations of Computer Science and Programming
### Summer 2026 | 16 Weeks
### Primary Resources
- CS50x: Harvard University's Introduction to Computer Science
- Code: The Hidden Language of Computer Hardware and Software by Charles Petzold

---

## Course Description
This course provides a rigorous introduction to computer science and programming using Harvard's CS50 as the primary instructional framework. Students will learn computational thinking, algorithms, data structures, memory, abstraction, and software design. Supplementary readings from *Code* by Charles Petzold explain how electricity, logic gates, binary arithmetic, and hardware architecture give rise to modern computing systems.

By the end of the course, the student will understand both how to program and how computers function from the physical level up through software.

---

## Learning Outcomes
By the end of this course, the student will be able to:
- Explain how computers represent information using binary.
- Describe the relationship between hardware and software.
- Write programs in C and Python.
- Implement fundamental algorithms and data structures.
- Use debugging and memory analysis tools.
- Build a complete final software project.

---

## Weekly Schedule

### Weeks 1–2: Foundations of Computing
- CS50 Week 0: Scratch
- Code Chapters 1–10
- Binary, logic, information representation

### Weeks 3–4: Programming in C
- Variables, conditionals, loops, functions
- Arrays and strings
- Code Chapters 11–20

### Weeks 5–6: Algorithms and Memory
- Searching and sorting
- Pointers and dynamic memory
- Debugging with gdb and valgrind

### Weeks 7–8: Data Structures
- Linked lists
- Hash tables
- Tries

### Weeks 9–10: Python Programming
- Syntax and scripting
- File I/O
- Automation

### Weeks 11–12: Databases and SQL
- Relational models
- Queries and normalization

### Weeks 13–14: Web Development
- HTML, CSS, Flask
- APIs and JSON

### Weeks 15–16: Final Project
- Design, implementation, documentation, presentation

---

## Major Assignments
- Weekly CS50 Problem Sets
- Programming exercises in C and Python
- Notes and summaries from *Code*
- Final programming project

# SYS 101 — Linux Systems Administration and Operating Systems
### Summer 2026 | 16 Weeks
### Primary Resource
- How Linux Works: What Every Superuser Should Know by Brian Ward

---

## Course Description
This course introduces Linux system administration and operating system fundamentals through practical study and extensive hands-on laboratory work. Students will learn how Linux boots, manages processes, handles storage, controls permissions, provides networking, and supports automation.

The course is taught entirely through direct interaction with Linux systems, using Ubuntu and Proxmox-based homelab infrastructure.

---

## Learning Outcomes
By the end of this course, the student will be able to:
- Navigate and administer Linux systems from the command line.
- Understand kernel, process, and memory fundamentals.
- Manage users, permissions, and services.
- Configure disks, filesystems, and mounts.
- Write Bash scripts and automate tasks.
- Secure and monitor production systems.

---

## Weekly Schedule

### Weeks 1–2: Linux Fundamentals
- Shell usage
- Filesystem hierarchy
- Basic commands

### Weeks 3–4: Devices, Processes, and Services
- /proc and /dev
- systemd
- Process control

### Weeks 5–6: Users and Permissions
- chmod, chown, sudo
- Group administration

### Weeks 7–8: Storage and Filesystems
- Partitions
- ext4
- fstab
- Swap

### Weeks 9–10: Boot and Package Management
- GRUB
- system initialization
- apt repositories

### Weeks 11–12: Networking and Security
- SSH hardening
- Firewalls
- Logging
- Fail2Ban

### Weeks 13–14: Shell Scripting and Automation
- Bash
- cron
- rsync backups

### Weeks 15–16: Monitoring and Production Operations
- Grafana and Prometheus
- Backup and disaster recovery

---

## Laboratory Projects
- Key-based SSH setup
- Automated nightly restart script
- Backup automation
- Storage server configuration
- Reverse proxy deployment
- Monitoring stack

# NET 101 — Internet Routing and Network Architecture
### Summer 2026 | 16 Weeks
### Primary Resource
- Internet Routing Architectures by Sam Halabi and Danny McPherson

---

## Course Description
This course introduces modern networking with an emphasis on how the global Internet is structured and routed. Students study IP addressing, routing, switching, autonomous systems, and the Border Gateway Protocol (BGP), while applying concepts directly to home and enterprise networks.

The course bridges theoretical networking concepts with practical homelab implementation.

---

## Learning Outcomes
By the end of this course, the student will be able to:
- Explain how data traverses networks and the Internet.
- Configure and troubleshoot IPv4 networking.
- Perform subnetting calculations.
- Understand routing protocols and autonomous systems.
- Describe BGP peering and path selection.
- Design segmented and secure networks.

---

## Weekly Schedule

### Weeks 1–2: Internet Fundamentals
- Network models
- Autonomous systems
- Packet switching

### Weeks 3–4: TCP/IP and Addressing
- IPv4
- Subnetting
- NAT
- Ports

### Weeks 5–6: Routing and Switching
- Static routing
- Layer 2 vs Layer 3

### Weeks 7–8: VLANs and Segmentation
- Broadcast domains
- Trunking

### Weeks 9–10: DNS, DHCP, and Core Services
- Name resolution
- Dynamic addressing

### Weeks 11–12: Firewalls and VPNs
- UFW/pfSense
- WireGuard

### Weeks 13–14: BGP Fundamentals
- Peering
- Route advertisements
- Path selection

### Weeks 15–16: Network Architecture Project
- Complete homelab network design

---
